# 🔍 QueryWise — AI-Powered SQL Intelligence Platform

QueryWise is a production-ready Streamlit application that lets you query any MySQL database using plain English. It uses Retrieval-Augmented Generation (RAG) to ground an LLM (Llama 3.3 via Groq) in your real database schema, ensuring accurate, hallucination-free SQL queries.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🧠 Natural Language SQL | Describe what you need; get production-ready MySQL queries |
| 🗂️ Schema Explorer | Visual table browser with row counts and 5-row previews |
| 💬 Conversational Chat | Multi-turn dialogue with memory for follow-up questions |
| ⚡ Auto-Execute | Optionally run generated SQL and display results instantly |
| 📜 Query History | Review and re-run all queries from the current session |
| 🌙 Dark / Light Mode | Toggle between polished dark and light themes |
| 🔒 Secure Credentials | API key and DB password never stored outside the session |

## 🚀 Quick Start

### 1. Prerequisites

- Python 3.10+
- MySQL server running locally or remotely
- [Groq API key](https://console.groq.com) (free tier available)

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Run the app

```bash
streamlit run app.py
```

### 4. Connect in the UI

1. Enter your **Groq API key** in the sidebar
2. Fill in your **MySQL credentials** (host, user, password, database)
3. Click **⚡ Connect to Database**
4. Once connected, explore your schema and start chatting!

## ⚙️ Configuration

### Environment variables (optional)

You can pre-fill credentials via environment variables to avoid typing them every time:

```bash
export GROQ_API_KEY="gsk_..."
export DB_HOST="localhost"
export DB_USER="root"
export DB_PASSWORD="yourpassword"
export DB_NAME="ecommerce_db"
```

> The app reads these on startup if present. They are never written to disk.

### Advanced settings (sidebar)

| Setting | Default | Description |
|---|---|---|
| Schema context chunks | 4 | Number of RAG chunks retrieved per query |
| Auto-execute SQL | Off | Automatically run and display query results |
| Always show formatted SQL | On | Pretty-print SQL in every assistant response |

## 🏗️ Architecture

```
User Query
    │
    ▼
ChromaDB (Ephemeral)          ← Schema strings embedded with all-MiniLM-L6-v2
    │  Top-K relevant chunks
    ▼
Groq LLM (Llama 3.3 70B)     ← Grounded with schema context
    │  Raw SQL / Chat response
    ▼
sqlparse formatter
    │
    ▼
Streamlit UI  ──(optional)──▶  MySQL execute → DataFrame results
```

## 🔐 Security Notes

- Database credentials and API keys are stored only in Streamlit's `session_state` for the duration of the browser session.
- Only **SELECT** queries are generated by the LLM (enforced via prompt rules).
- The auto-execute feature does **not** run any non-SELECT statements.
- Do **not** commit `.streamlit/secrets.toml` with real credentials to version control.

## 🛠️ Extending QueryWise

### Adding a new LLM provider

Edit `utils/llm.py` and swap `ChatGroq` for any LangChain-compatible chat model (OpenAI, Anthropic, Ollama, etc.).

### Supporting other databases

Edit `utils/db.py` to replace `mysql.connector` with `psycopg2` (PostgreSQL), `pyodbc` (SQL Server), or any other DBAPI-compatible connector.

### Persisting the vector store

Replace `chromadb.EphemeralClient()` in `utils/vector_store.py` with `chromadb.PersistentClient(path="./chroma_data")` to keep embeddings across restarts.
